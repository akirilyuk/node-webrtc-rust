Local Sherpa — three clients, one room

Open three browser tabs to this URL (add ?slot=1, ?slot=2, ?slot=3 to label logs). Each tab is one client-* peer → one VoiceAgent → shared Sherpa STT/TTS models in the Node process. When you speak in a tab, TTS replies go only to that tab. Use Speak to all to broadcast TTS to every connected tab.

Server setup

  1. Download models via voice-agent-local-sherpa (download-stt, download-tts)
  2. Export SHERPA_STT_MODEL_PATH and SHERPA_TTS_MODEL_PATH
  3. Start: npm run start --workspace=@node-webrtc-rust/example-voice-agent-local-sherpa-multi-client
  4. Optional cap: npm run start:cap-2 --workspace=@node-webrtc-rust/example-voice-agent-local-sherpa-multi-client

Three-server-session pattern

  1. Tab 1, 2, 3 — same room sherpa-multi, click Connect in each.
  2. Optional: under Connect, check Pause mic when this tab is in the background on tabs you are not speaking from (or ?slot=2&pauseMicBackground=1).
  3. Server log shows [voice client-tabN] — three VoiceAgents, one Sherpa model load.
  4. With VOICE_MAX_CONCURRENT_SESSIONS=2, the third tab should not get an offer.
  5. Check GET /api/capacity for active/max/rejected counts.
