Local Sherpa STT + TTS — Browser Client

Your microphone is sent to a Node VoiceAgent over WebRTC. Speech is transcribed and synthesized on-device with Sherpa-ONNX (no cloud API keys). Partial and final transcripts arrive on the voice-control data channel. Use the Speak form for local Piper TTS and barge-in demos.

Server setup (one time)

  1. Download STT weights: npm run download-stt --workspace=@node-webrtc-rust/example-voice-agent-local-sherpa
  2. Download TTS voice: npm run download-tts --workspace=@node-webrtc-rust/example-voice-agent-local-sherpa
  3. Export SHERPA_STT_MODEL_PATH and SHERPA_TTS_MODEL_PATH (printed by the scripts)
  4. Start this server: npm run start --workspace=@node-webrtc-rust/example-voice-agent-local-sherpa

Full guide: examples/voice-agent-local-sherpa/README.md

Test local STT

  1. Click Connect and allow microphone access.
  2. Watch the Your microphone waveform — it should move when you speak.
  3. Speak clearly — watch user_speech_partial update in the log.
  4. Pause briefly — Sherpa endpointing emits user_speech_final.

Latency depends on CPU and model size. The default English Zipformer bundle is tuned for streaming voice UI (~100–300 ms partials on modern laptops).
