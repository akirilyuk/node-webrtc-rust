Multi-Session Voice Pod

One Node pod serves many concurrent calls. Each session ID is an independent signaling room; the pod spawns one VoiceAgent per WebRTC connection and tears it down when you disconnect.

Try multiple sessions

  1. Start the pod: npm run start --workspace=@node-webrtc-rust/example-voice-agent-multi-session-pod
  2. Open this page in tab A with session call-1, connect, speak.
  3. Open tab B with session call-2 — both run on the same pod.
  4. Disconnect tab A — watch the server log tear down that session's agent.
